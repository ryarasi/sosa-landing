# Contributing to SoSA

Welcome, Architect! We're building the infrastructure for human liberation, and every contribution matters.

## Our Philosophy

Before contributing, understand that SoSA is not just another project - it's the architecture of human freedom. Every line of code, every design decision, every contribution should align with our core principles:

- **Zero Centralized Control**: No single person or group should gain controlling influence
- **Radical Transparency**: All decisions and code must be auditable forever
- **Freedom Through Contribution**: Your worth is determined by your contribution, not your position
- **Continuous Evolution**: We improve daily, refine constantly, evolve perpetually

## How to Contribute

### 1. Understand the Vision
Read our [vision document](docs/vision.md) and [manifest](docs/manifest.md) to understand what we're building and why.

### 2. Find Your Area
- **Code Architecture**: Build the technical infrastructure
- **Documentation**: Help others understand and contribute
- **Research**: Provide insights for policy recommendations
- **Design**: Create interfaces that embody sovereignty
- **Community**: Help coordinate our distributed governance

### 3. Before You Start
- Check existing issues and pull requests
- Discuss major changes in an issue first
- Ensure your contribution aligns with our vision

### 4. Making Changes

#### Setup
```bash
git clone https://github.com/ryarasi/sosa-landing.git
cd sosa-landing
npm install
npm run dev
```

#### Code Standards
- Write clear, self-documenting code
- Follow existing patterns and conventions
- Test your changes thoroughly
- Ensure no single point of failure

#### Commit Messages
- Use clear, descriptive commit messages
- Reference issues when applicable
- Sign your commits to maintain accountability

### 5. Submitting Changes
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-contribution`)
3. Make your changes
4. Test thoroughly
5. Commit with a clear message
6. Push to your fork
7. Open a Pull Request

## Pull Request Guidelines

Your PR should:
- Have a clear title and description
- Reference any related issues
- Include tests if adding functionality
- Update documentation as needed
- Align with our core principles

## Code of Conduct

### We Are Architects of Freedom
- Treat every contributor as a sovereign individual
- Respect diverse perspectives and approaches
- Focus on building, not destroying
- Collaborate through merit, not politics

### We Reject
- Centralized control or gatekeeping
- Hidden agendas or backdoor dealings
- Discrimination based on anything except contribution quality
- Actions that compromise user sovereignty

## Recognition

Contributors are recognized through:
- Transparent contribution tracking
- Merit-based influence in governance
- Attribution in our decentralized systems
- The intrinsic reward of building human freedom

## Questions?

- Open an issue for technical questions
- Join our Discord for community discussions
- Read the documentation for guidance

Remember: You're not just contributing code - you're architecting the future of human coordination.

**Welcome to your sovereignty.**
